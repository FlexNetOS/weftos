**EXPERIMENTAL (0.6.10)** вЂ” Iteration 2 of an EML-Transformer. Ships behind the `experimental-attention` feature on `eml-core`. Default builds are unchanged.

EML Attention is WeftOS's first step toward replacing backprop-trained attention with a gradient-free, coordinate-descent-trained, weight-snapping primitive that lives in the same substrate as every other learned function in the kernel. The classical path (`EmlModel` wrappers for scalar-output learned heuristics вЂ” see [EML вЂ” Self-Learning Functions](/weftos/eml)) continues to run unchanged.

**Feature**: `experimental-attention` (off by default)
**Source**: `crates/eml-core/src/attention.rs` (13 tests)
**Phase**: experimental; part of the 0.6.x roadmap, not a production primitive
**Related**: [EML вЂ” Self-Learning Functions](/weftos/eml), [ECC Cognitive Substrate](/weftos/ecc)

## Why

Every learned function in WeftOS today is a scalar-output `EmlModel`: `QueryFusionModel`, `CausalCollapseModel`, `SurpriseScorer`, `GossipTiming`, etc. These work brilliantly for isolated thresholds and scoring formulas, but they don't compose. A real cognitive primitive needs **attention** вЂ” content-addressable routing across a sequence вЂ” which no scalar EML head can express.

The theoretical bridge is Odrzywolel 2026's proof that the EML operator `eml(x, y) = exp(x) - ln(y)` combined with the constant `1` can reconstruct every elementary function. Attention is built from those elementary functions (matrix multiplication, softmax, affine projection). So attention is in principle expressible in EML. This page documents the pragmatic first step.

## What Iteration 0 is

A single-head attention block at toy scale вЂ” `d_model в‰¤ 32`, `seq_len в‰¤ 8`, per-submodel depth 3-5 вЂ” built from five `EmlModel` instances:

| Submodel | Shape | Role |
|----------|-------|------|
| `q_model` | `(seq_lenВ·d_model) в†’ (seq_lenВ·d_k)` | Query projection |
| `k_model` | `(seq_lenВ·d_model) в†’ (seq_lenВ·d_k)` | Key projection |
| `v_model` | `(seq_lenВ·d_model) в†’ (seq_lenВ·d_k)` | Value projection |
| `softmax_model` | `seq_len в†’ seq_len` | Learned row-softmax approximator |
| `out_model` | `(seq_lenВ·d_k) в†’ (seq_lenВ·d_model)` | Output projection |

The inter-projection matmuls (QВ·KбµЂ and AВ·V) are `f64` in Iteration 0. The dev note [`eml_model_development.md`](https://github.com/weave-logic-ai/weftos/blob/master/.planning/development_notes/eml_model_development.md) is explicit that hybrid EML + float is acceptable through Iteration 4+; we adopt the hybrid from day one to make the spike tractable.

Training is **gradient-free**. `record(input, target)` stores end-to-end pairs in an internal buffer (cap 256). `train()` runs a forward pass on the buffer, derives per-submodel targets via self-distillation вЂ” numerical softmax for `softmax_model`; `(context, target)` for `out_model` вЂ” and trains using the existing coordinate-descent loop with random restarts.

## The 4-phase benchmark

Mirrors the layout used by `clawft-weave/src/commands/bench_cmd.rs` (Warmup в†’ Transport в†’ Compute в†’ Scalability) adapted for learned functions:

- **Phase 1 вЂ” Warmup**: single forward-pass timing + `to_json`/`from_json` roundtrip
- **Phase 2 вЂ” Convergence**: train on a memorizable identity task (96 synthetic samples); report converged flag, final MSE, training-round count
- **Phase 3 вЂ” Compute**: inference latency over 256 random inputs; mean and p99
- **Phase 4 вЂ” Scalability**: sweep `(seq_len, d_model)` across `{(4,8), (4,16), (8,8), (8,16)}`; report param count and mean inference per point

Results are returned as a single `AttentionBenchmark` struct serializable as JSON вЂ” the benchmark's primary use is quantifying changes between iterations.

```rust
use eml_core::run_benchmark;

let bench = run_benchmark(
    /* d_model */ 8,
    /* d_k     */ 4,
    /* seq_len */ 4,
    /* depth   */ 3,
)?;
println!("param_count = {}", bench.param_count);
println!("phase3_mean = {} ns", bench.phase3_inference_ns_mean);
println!("phase3_p99  = {} ns", bench.phase3_inference_ns_p99);
```

## Go/no-go criteria

The spike is considered successful if all five hold:

1. Converges on the identity task in в‰¤ 3 training rounds
2. Inference p99 в‰¤ 5 Вµs at `(seq_len=4, d_model=8, d_k=4, depth=3)`
3. 1000 forward passes without NaN / Inf at the default shape
4. Serialization roundtrip preserves shape and parameter count
5. Phase-4 scaling shows polynomial-not-exponential growth up to `(seq_len=8, d_model=16)`

If Iteration 0 fails any criterion we stop and log the substrate limit; we do not proceed to Iteration 1 until this gate passes.

## Usage

```rust
use eml_core::ToyEmlAttention;

let mut attn = ToyEmlAttention::new(
    "demo",
    /* d_model */ 8,
    /* d_k     */ 4,
    /* seq_len */ 4,
    /* depth   */ 3,
)?;

// Forward pass on random input
let x = vec![0.5; 4 * 8];
let y = attn.forward(&x)?;
assert_eq!(y.len(), 4 * 8);

// Record + train
for i in 0..32 {
    let input = (0..32).map(|j| ((i * j) as f64).sin()).collect::<Vec<_>>();
    attn.record(input.clone(), input)?; // identity task
}
let converged = attn.train();
```

## Honest limitations

- **No end-to-end coordinate descent yet**. Q/K/V models stay at default init in Iteration 0; only `softmax_model` and `out_model` train. This validates the architecture but not the full training story. Iteration 1 introduces end-to-end coordinate descent over the union of all five sub-models.
- **Hybrid, not pure EML**. The two matmuls are `f64`. Pure-EML matmul is a research problem (unsigned `ln` requirements); the dev note explicitly treats hybrid as acceptable through Iteration 4+.
- **Toy scale only**. `d_model в‰¤ 32`, `seq_len в‰¤ 8`. This is a *deliberate* ceiling on Iteration 0 вЂ” the scaling plan grows these over four more iterations.
- **Not yet wired into any production path**. No existing EML wrapper has been replaced by this block. That's a Sprint 18+ concern.

## How does this compare to a plain attention?

`eml-core` also ships a `BaselineAttention` вЂ” same public API, same
training loop, but the Q/K/V/out projections are plain `WВ·x + b` affine
maps with no EML tree. Trained with the exact same gradient-free
coordinate-descent optimizer, same trial budget, same seed.

At the gate shape `(d_model=4, d_k=2, seq_len=2, depth=3)`, 5000 trials
Г— 3 rounds:

| Metric | EML (SafeTree) | Baseline (affine) |
|---|---|---|
| Parameters | 352 | 148 |
| Baseline MSE (untrained) | 1.48 | 0.23 |
| Final MSE | 1.54 | 0.055 |
| MSE reduction | -4% | 76% |
| Forward-pass p99 | 1.2 Вµs | 0.36 Вµs |

At this scale, plain affine attention is strictly better on convergence,
latency, and parameter efficiency. What EML buys is:

- **Weight snapping** вЂ” trained EML parameters converge to exact integers
  at Level 0; the model is expressible as a closed-form mathematical
  expression
- **ExoChain-auditable gradient-free training** вЂ” every training step is
  a deterministic coordinate-descent move, recorded in the kernel event
  stream, no backprop / autograd / magic constants
- **Substrate alignment** вЂ” the 12 existing EML wrappers
  (`QueryFusionModel`, `CausalCollapseModel`, `SurpriseScorer`, etc.)
  use the same primitives, so attention slots into the learned-function
  story without introducing a second optimization paradigm

These are real-but-narrow wins. The measurement is shipped so you can
make the tradeoff explicitly. If convergence speed or latency are
primary, plain affine attention wins today.
